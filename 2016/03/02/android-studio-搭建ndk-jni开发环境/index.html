android studio 搭建ndk jni开发环境

发表于 2016-03-02   |  

下载ndk

官网下载地址 下载地址 （我用的最新的，貌似之前用的旧版本一直有问题,建议还是下载最新的，我用的是 android-ndk32-r10b-windows-x86_64.zip）

##新建android项目
代码很简单

在main目录新建jni文件夹，Alt+Enter快捷键,你会发现android studio

配置项目ndk.dir

打开项目根目录local.properties
sdk.dir=D\:\\sdk ndk.dir=D\:\\android-ndk-r10b
如下图：

或者打开File(菜单栏)–>Project Structure–>Android NDK Location 进行设置 如下图:

配置gradle.propertis

在build.gralde 同级目录新建 gradle.propertis文件 文件内容为：
android.useDeprecatedNdk=true

否则会报以下的错误

Error:(13, 0) Error: NDK integration is deprecated in the current plugin. Consider trying the new experimental plugin. For details, see http://tools.android.com/tech-docs/new-build-system/gradle-experimental. Set “android.useDeprecatedNdk=true” in gradle.properties to continue using the current NDK integration.
Open File

##配置build.gradle
在build.gradle配置ndk 如以下代码
defaultConfig { applicationId "com.sanniu.testjni" minSdkVersion 15 targetSdkVersion 23 versionCode 1 versionName "1.0" ndk { moduleName "jniTest" // <-- This is the name of my C++ module! ldLibs "log", "z", "m" abiFilters "armeabi", "armeabi-v7a", "x86" } }

开始编译

Bulid–>Make Project 会在build\intermediates\ndk\debug 这个目录生成相关的编译后的文件

配置打包编译后的so文件到lib目录，配置build.gradle
sourceSets { main { jniLibs.srcDirs = ['libs'] } }
我本人的build.gradle完整文件为：
`apply plugin: ‘com.android.application’

android {
compileSdkVersion 23
buildToolsVersion “23.0.2”

defaultConfig {
    applicationId "com.sanniu.testjni"
    minSdkVersion 15
    targetSdkVersion 23
    versionCode 1
    versionName "1.0"
    ndk {
        moduleName "jniTest" // <-- This is the name of my C++ module!
        ldLibs "log", "z", "m"
        abiFilters "armeabi", "armeabi-v7a", "x86"
    }
}
buildTypes {
    release {
        minifyEnabled false
        proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
    }
}
sourceSets {
    main {
        jniLibs.srcDirs = ['libs']
    }
}

}

dependencies {
compile fileTree(dir: ‘libs’, include: [‘*.jar’])
testCompile ‘junit:junit:4.12’
compile ‘com.android.support:appcompat-v7:23.1.1’
}
`

其他相关问题解决

编译运行项目可能还会有相关下面的问题
`Error:Execution failed for task ‘:app:compileDebugNdk’.

com.android.ide.common.process.ProcessException: org.gradle.process.internal.ExecException: Process ‘command ‘D:\android-ndk-r10\ndk-build.cmd’’ finished with non-zero exit value 2网上的解决办法是： 在jni新建一个空白的empty.c文件 编译即可 如果还编译不通过可以使用android{

sourceSets.main {
    jni.srcDirs = []
}

} `
